# Ibraim Library

Персональный сайт-архив для артиста: личный Spotify-style плеер, дневник процесса, тёмный минималистичный интерфейс. Загружать музыку может только владелец через защищённую паролем админку.

```
 ┌─ sidebar ─┬──── контент ─────────────────────────┐
 │ Home      │  крупная обложка последнего релиза   │
 │ Music     │  ▶ Play                              │
 │ Demos     │  подборки: релизы, демки, старое,    │
 │ Posts     │  невышедшее, любимое                 │
 │ About     │  записи из дневника                  │
 └───────────┴──────────────────────────────────────┘
   плеер внизу (как в Spotify)
```

## Старт

```bash
npm install
npm run dev          # http://localhost:3000
```

Production:
```bash
npm run build && npm start
```

## Что есть

**Публичная часть**
- `/` — главная: крупный hero последнего релиза, подборки «Последний релиз», «Демки», «Старое», «Невышедшее», «Любимое», лента из дневника
- `/music` — все треки с фильтрами (релизы / архив / скоро / любимое)
- `/demos` — только демки
- `/posts` — дневник: видео из студии, тексты, демки, сниппеты, новости, заметки
- `/about` — обо мне + статистика библиотеки + ссылки
- `/track/[id]` — страница трека: обложка, плеер, описание, личная заметка, текст, вложения, похожее
- `/post/[id]` — страница записи с медиа (фото, видео, аудио)
- сквозной плеер внизу, очередь, перемотка, громкость, эквалайзер на воспроизводимом треке

**Админка** — только для тебя
- `/admin/login` — вход по паролю
- `/admin` — дашборд
- `/admin/tracks` — список, `/admin/tracks/new` — загрузка трека (аудио, обложка, описание, заметка, текст, жанр, настроение, статус: релиз/демо/архив/скоро, дата, отметка «любимое»)
- `/admin/posts` — записи дневника с любым количеством вложений
- `/admin/settings` — имя, описание, ссылки, смена пароля

## Пароль

Стандартный пароль — **`ibraim`**. Поменять можно двумя способами:

1. В админке: `Admin → Settings → пароль`
2. Через переменную окружения `ADMIN_PASSWORD` (применяется при первой загрузке, до создания `data/db.json`)

Сессия — подписанный JWT в httpOnly cookie. Секрет берётся из `SESSION_SECRET` (по умолчанию — локальный).

## Где что хранится

- `data/db.json` — данные (треки, посты, настройки, хеш пароля). Создаётся при первом запуске.
- `public/uploads/` — все загруженные файлы (mp3, обложки, видео, фото).

Обе папки в `.gitignore` — это твой локальный архив, а не часть репозитория.

## Стек

Next.js 14 (App Router, server actions) · TypeScript · Tailwind · JSON-storage · bcryptjs + jose для сессий. Без базы данных и сторонних сервисов — всё локально, лежит в одном репозитории.
